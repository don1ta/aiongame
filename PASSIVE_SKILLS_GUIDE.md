# passive_skills.json 填寫規則

## 基本結構

```json
{
    "職業名稱": {
        "技能名稱": {
            "屬性名稱": 數值
        }
    }
}
```

---

## 核心規則

### 規則 1：整數型屬性 → 直接填整數

這些屬性在遊戲中是「固定數值」，填整數即可。

```json
"生命力": 200,
"精神力": 150,
"命中": 100,
"迴避": 200,
"暴擊": 100,
"格擋": 200,
"異常狀態抵抗": 16.5,
"生命力自然恢復": 20,
"精神力自然恢復": 5
```

### 規則 2：百分比型屬性 → 填小數（÷100）

這些屬性在遊戲中顯示為 `%`，填寫時用小數表示。

```json
"傷害增幅": 0.10,        // 遊戲顯示 10%
"暴擊傷害增幅": 0.06,    // 遊戲顯示 6%
"生命力增加": 0.07,      // 遊戲顯示 7%（最大生命力增加）
"精神力增加": 0.07,      // 遊戲顯示 7%
"防禦力增加": 0.02,      // 遊戲顯示 2%（防禦力百分比增加）
"強擊": 0.03,            // 遊戲顯示 3%
"完美": 0.05,            // 遊戲顯示 5%
"鐵壁": 0.01,            // 遊戲顯示 1%
"後方傷害增幅": 0.06,    // 遊戲顯示 6%
"衝擊系擊中增加": 0.12,  // 遊戲顯示 12%
"精神系擊中增加": 0.11,  // 遊戲顯示 11%
"肉體系抵抗": 0.06,      // 遊戲顯示 6%
"精神系抵抗增加": 0.115, // 遊戲顯示 11.5%
"衝擊系抵抗增加": 0.115, // 遊戲顯示 11.5%
"治癒增幅增加": 0.10,    // 遊戲顯示 10%
"生命力恢復增加": 0.06   // 遊戲顯示 6%（生命力藥水恢復）
```

## ⚠️ 常見錯誤

### ❌ 錯誤：用「增加」後綴命名百分比屬性

```json
// ❌ 錯誤 - 和裝備的 key 不一樣，無法合併計算
"傷害增幅增加": 0.10,
"暴擊抵抗增加": 100,
"異常狀態抵抗增加": 0.165
```

```json
// ✅ 正確
"傷害增幅": 0.10,
"暴擊抵抗": 100,       // 暴擊抵抗是整數型！
"異常狀態抵抗": 16.5   // 異常狀態抵抗是整數型！
```

### ❌ 錯誤：整數型屬性填小數

```json
// ❌ 錯誤 - 防禦力是整數型，0.02 會污染整個防禦力計算
"防禦力": 0.02
```

```json
// ✅ 正確 - 百分比增加用「防禦力增加」
"防禦力增加": 0.02
```

## 判斷方法

**不確定是整數還是百分比？**

看遊戲技能描述：
- 顯示 `+200` → 整數型，直接填 `200`
- 顯示 `+7%` → 百分比型，填 `0.07`

**不確定 key 名稱？**

用遊戲技能描述中的屬性名稱，去掉「額外」、「最大」等前綴，保留核心名稱。
